
Presuming you've logged in to the NeoLoad CLI...

neoload-compose or neoloadc (alias)

2020:

neoloadc transaction One http --get http://httpbin.org/get?test=123 delay 1s \
         transaction Two http --post http://httpbin.org/post --body '${var}' delay 250ms \
         header 'accept=application/json' \
         ramp --to 10 --per 1s \
         duration 2m current \
         run --zone $NLW_ZONE_STATIC MyTest

# continuation, particularly when reading large contents from STDIN
neoloadc transaction One http --get http://httpbin.org/get?test=123 delay 1s
cat ~/some.json | neoloadc --continue transaction Two http --post http://httpbin.org/post --body - delay 250ms \

# add a header, and with --all apply to all prior builder requests
neoloadc -c header --all --name "accept" --value "application/json"
...

# add SLAs; by default, applies to most recent request or container only, unless --all is used
neoloadc -c sla --all --threshold error-rate --warn 5 --fail 10 my_sla
# apply a specific sla (defined up front or prior) to a request or container
neoloadc -c transaction One http --get http://httpbin.org/get?test=123 --sla some_specific_sla delay 1s

# at any time, you can see what the yaml would be sent to NeoLoad Web
neoloadc current

# run command makes system calls to standard NeoLoad CLI ('neoload ...')
neoloadc run --save --zone $NLW_ZONE_STATIC
...or...
neoloadc config zone $NLW_ZONE_STATIC test-setting MyTest
neoloadc run
